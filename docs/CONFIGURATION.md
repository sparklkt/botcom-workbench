# Configuration

BotCom Workbench is configured by environment variables. It does not require a cloud account to start.

## Paths

Default values:

```text
BOTCOM_HOME=~/BotCom
BOTCOM_MEDIA_OPS_ROOT=~/BotCom/media-ops
BOTCOM_WORKBENCH_ROOT=~/BotCom/AI-Workbench
```

Use custom paths when your operating workspace or first content-operations adapter lives elsewhere:

```bash
export BOTCOM_HOME="$HOME/BotCom"
export BOTCOM_MEDIA_OPS_ROOT="$HOME/BotCom/media-ops"
export BOTCOM_WORKBENCH_ROOT="$HOME/BotCom/AI-Workbench"
```

## Ports

```text
BOTCOM_WORKBENCH_PORT=4570
MEDIA_OPS_REVIEW_PORT=8787
BOTCOM_RELEASE_REPO=
```

`BOTCOM_WORKBENCH_PORT` controls the local backend port used by both the Electron wrapper and the web server.

`BOTCOM_RELEASE_REPO` is optional. When empty, release checking is disabled. When set, use `owner/repo`.

## Review server token

BotCom Workbench reads the review token from either:

```text
MEDIA_OPS_REVIEW_TOKEN
```

or a local `media-ops/runtime/mobile_entry.md` file generated by the review server.

The token is sent only to the local review server through the backend proxy. It is not returned to browser API callers.

## Feishu / Lark

```text
FEISHU_BOT_WEBHOOK=
FEISHU_BOT_SECRET=
LARK_BOT_WEBHOOK=
LARK_BOT_SECRET=
```

## WeCom / Enterprise WeChat

```text
WECHAT_WORK_BOT_WEBHOOK=
WECHAT_BOT_WEBHOOK=
WECHAT_WEBHOOK=
```

## Local env files

The notification bridge reads, in order:

```text
$BOTCOM_MEDIA_OPS_ROOT/.env
./.env
process environment
```

Do not commit `.env`.
